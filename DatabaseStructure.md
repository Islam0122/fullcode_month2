# 🗄 Database Structure — Finance Bot

### 1️⃣ Таблица `users`

| Поле       | Тип        | Описание                                |
| ---------- | ---------- | --------------------------------------- |
| id         | INTEGER PK | Автоинкремент, уникальный идентификатор |
| user_id    | INTEGER    | Telegram user_id                        |
| created_at | DATETIME   | Дата регистрации пользователя           |

**Примечания:**

* Сохраняем каждого пользователя при первом запуске `/start`.
* Здесь нет поля `balance`, он считается из `transactions`.

---

### 2️⃣ Таблица `transactions`

| Поле        | Тип        | Описание                                                               |
| ----------- | ---------- | ---------------------------------------------------------------------- |
| id          | INTEGER PK | Автоинкремент, уникальный идентификатор                                |
| user_id     | INTEGER    | Telegram user_id, связывает транзакцию с пользователем                 |
| type        | TEXT       | `'income'` или `'expense'`                                             |
| amount      | REAL       | Сумма дохода или расхода                                               |
| category    | TEXT       | Категория расхода (например: Еда, Транспорт, Покупки). NULL для дохода |
| description | TEXT       | Текстовое описание операции                                            |
| created_at  | DATETIME   | Дата и время создания транзакции                                       |

**Примечания:**

* Для доходов поле `category` остаётся пустым (`NULL`).
* Тип транзакции (`type`) обязателен для вычисления баланса.
